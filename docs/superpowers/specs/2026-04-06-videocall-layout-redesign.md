# 视频通话页面布局重新设计 - 设计规格

## 问题背景

原有视频通话页面在底部放置了三个大按钮（声音、摄像头、退出登录），占据了过多视觉空间，且没有专门的对话内容显示区域。需要重新布局，让对话字幕内容可以铺满页面，控制栏放在底部。

## 设计目标

- 重新排列控件位置：退出登录放右上角，字幕放左上角
- 对话字幕内容区域铺满页面主体
- 底部控制栏：左侧小的声音按钮，中间长输入框，右侧小的摄像头按钮，加上发送按钮
- 所有元素风格统一（圆角、半透明、毛玻璃）
- 边距对齐一致

## 最终布局设计

### 整体布局结构

```
┌─────────────────────────────────────────────┐
│ [字幕]                        [退出登录]      │ ← 顶部角落按钮
├─────────────────────────────────────────────┤
│                                             │
│                                             │
│  对话内容区域（铺满主体）：                   │
│  [模型回复 1]                    [用户输入 1] │
│                                             │
│  [模型回复 2]                    [用户输入 2] │
│                                             │
│             [状态提示：根据状态显示文字]      │ ← 底部状态条
│                                             │
├─────────────────────────────────────────────┤
│ [🎤]   [输入对话...]          [发送] [📷]    │ ← 底部控制栏
└─────────────────────────────────────────────┘
```

### 各区域详细设计

#### 1. 顶部角落按钮

| 位置 | 控件 | 样式 |
|------|------|------|
| 左上角 | 字幕开关 | `rgba(0,0,0,0.6)` 背景，白色文字，圆角 9999px，12px 上下内边距，20px 左右内边距，毛玻璃 |
| 右上角 | 退出登录 | `rgba(220, 38, 38, 0.95)` 背景，白色文字，圆角 9999px，尺寸同上，毛玻璃 |

- 左右边距：24px，顶部边距：16px

#### 2. 背景层

- 视频内容铺满整个页面（`inset: 0`）
- 半透明黑色遮罩，让前景对话内容更清晰

#### 3. 对话内容区域

- 位置：`inset: 70px 24px 90px 24px`（顶部留出角落按钮空间，底部留出控制栏空间）
- 背景：`rgba(0,0,0,0.4)`，圆角 16px，毛玻璃
- 布局：flex 纵向排列
  - 上部：可滚动区域，显示对话历史
    - 模型回复：居左，蓝色气泡 `rgba(59, 130, 246, 0.9)`
    - 用户输入：居右，绿色气泡 `rgba(34, 197, 94, 0.9)`
  - 下部：状态提示条
    - 背景：`rgba(0,0,0,0.6)`，圆角 12px，居中文字
    - 根据连接和语音状态显示不同提示文字：
      - 未连接后端：显示「未连接」
      - 已连接但未开始对话：显示「问问我」
      - 正在语音识别：显示「正在听」
      - 等待模型回复：显示「思考中」
      - 模型正在生成回复：显示「正在生成回复」

#### 4. 底部控制栏

- 位置：`bottom: 24px; left: 24px; right: 24px`
- 布局：flex 横向排列，间隙 12px

| 位置 | 控件 | 尺寸 | 样式 |
|------|------|------|------|
| 最左 | 麦克风开关 | 宽 48px，高 48px | 圆形，绿色背景 `rgba(22, 163, 74, 0.95)`，麦克风图标，毛玻璃 |
| 中间 | 输入框 | flex: 1（占据剩余空间），最小高度 48px | 浅灰色背景 `rgba(255, 255, 255, 0.92)`，圆角 9999px，水平内边距 20px，提示文字「输入对话...」，毛玻璃 |
| 中右 | 发送按钮 | 自适应文字宽度 | 蓝色背景 `rgba(59, 130, 246, 0.95)`，圆角 9999px，文字「发送」，毛玻璃 |
| 最右 | 摄像头开关 | 宽 48px，高 48px | 圆形，深灰色背景 `rgba(31, 41, 55, 0.95)`，摄像头图标，毛玻璃 |

## 样式规范

- 所有可点击控件都使用半透明背景 + `backdrop-filter: blur(8px)` 毛玻璃效果
- 圆角统一：大按钮使用 `9999px`（胶囊圆角），对话区域使用 `16px`，状态条使用 `12px`
- 颜色保持原项目风格：
  - 麦克风开启：绿色
  - 摄像头关闭：深灰色
  - 退出登录：红色
  - 发送按钮/模型气泡：蓝色
  - 用户气泡：绿色

## 现有组件复用

- 使用现有的 `useAudioCapture` 组合式函数处理麦克风状态
- 使用现有的 `useCameraCapture` 组合式函数处理摄像头状态
- 使用现有的 Pinia store 处理认证状态（退出登出）
- 使用现有的 WebSocket 客户端处理后端连接状态

## 文件修改

主要修改文件：`frontend/src/components/VideoCallPage.vue`

- 调整模板布局结构
- 更新 Tailwind CSS 类名
- 添加状态提示文字的逻辑
- 不改变现有功能逻辑，只调整UI布局

## 验收标准

1. 布局符合设计图：字幕按钮左上角，退出登录右上角
2. 对话内容区域铺满页面主体
3. 底部控制栏顺序正确：麦克风 - 输入框 - 发送 - 摄像头
4. 状态提示条根据不同状态显示正确文字
5. 所有元素样式风格统一，边距对齐一致
